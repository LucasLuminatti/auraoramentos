## Surface & Component Inventory

### UX-01 — Redirect empty-state in ProdutoAutocomplete

**Location:** `ProdutoAutocomplete.tsx` — replaces `<p>Nenhum produto encontrado</p>` (line 52) when results are empty AND a fallback query detects the code belongs to a different `tipo_produto`.

**Composition:**

```
div.px-3.py-3.space-y-2   ← replaces existing <p> when redirect detected
  p.text-sm.text-blue-900        ← "Este produto é um {tipo} — adicione em Sistemas de Iluminação"
  Button variant="outline" size="sm" className="gap-2 w-full text-xs"
    ArrowRight (lucide, h-3.5 w-3.5)
    "Ir para Sistemas de Iluminação"
```

Container: same `bg-popover` dropdown (`absolute z-50 mt-1 max-h-60 w-full overflow-auto rounded-lg border shadow-lg`) — do not change container styles.

**States:**

| State | What renders |
|-------|-------------|
| No query (< 2 chars) | Nothing — dropdown stays closed |
| Query with results | Normal results list (unchanged) |
| Query empty, no fallback needed | `<p className="py-4 text-center text-sm text-muted-foreground">Nenhum produto encontrado</p>` (unchanged) |
| Query empty, fallback detects tipo real | Blue redirect block with tipo name + "Ir para Sistemas de Iluminação" button |
| Fallback loading | Spinner (same `Loader2` pattern as main search) while fallback query runs |

**Tipo display names (pt-BR):**

| `tipo_produto` value | Display string |
|----------------------|---------------|
| `'perfil'` | "perfil" |
| `'fita'` | "fita LED" |
| `'driver'` | "driver" |

Full message: `"Este produto é um {tipo} — adicione em Sistemas de Iluminação"`

**Props change:** `ProdutoAutocomplete` gains optional `onRedirectToSistemas?: () => void`. When the button is clicked: call `onRedirectToSistemas()` and close the dropdown (`setOpen(false)`).

**How AmbienteCard wires it (D-02):** `AmbienteCard` owns a `activeTab` state (currently implicit in `Tabs defaultValue`). Upgrade to controlled: `const [activeTab, setActiveTab] = useState<'luminarias' | 'sistemas'>('luminarias')`. Pass `onRedirectToSistemas={() => setActiveTab('sistemas')}` to the Luminária tab's `ProdutoAutocomplete`. After tab switch, focus behavior: no auto-focus required (discretion per D-02 — tab switch alone satisfies the contract).

### RES-04 — Duplicar sistema (AmbienteCard)

**Location:** `AmbienteCard.tsx` — sistema header bar (the `div.flex.items-center.justify-between.px-4.py-2.bg-muted/40.border-b` containing "Sistema N" + badges + remove button).

**Composition:** Add a `Copy` (or `CopyPlus`) icon button next to the existing remove button, in the right-side actions group:

```tsx
<div className="flex items-center gap-2">   ← existing wrapper
  {subtotal > 0 && <Badge ...>Subtotal ...</Badge>}
  {/* NEW: Duplicate button */}
  <Button
    size="icon"
    variant="ghost"
    className="h-7 w-7 text-muted-foreground hover:text-foreground"
    title="Duplicar sistema"
    onClick={() => onDuplicateSistema(si)}
  >
    <Copy className="h-3.5 w-3.5" />
  </Button>
```

Icon: `Copy` from lucide-react (preferred). `CopyPlus` acceptable — executor's discretion per D-05 discretion note.

**Prop added to AmbienteCard:** none needed at this level — `duplicarSistema(si)` is a local handler inside `AmbienteCard` that calls `onChange` with the cloned array.

- `local` field: append `" (cópia)"` — e.g. `"Sanca"` → `"Sanca (cópia)"`.
- Cloned sistema inserted at `si + 1` (immediately after original).
- No dialog, no confirmation (D-07).

| State | Appearance |
|-------|-----------|
| Default | `text-muted-foreground`, no fill |
| Hover | `text-foreground` (via `hover:text-foreground`) |

### UX-04 — Duplicar ambiente (Step2Ambientes)

**Location:** `Step2Ambientes.tsx` — the `AmbienteCard` render loop. The remove button is inside `AmbienteCard` itself (passed via `onRemove`). For UX-04, the duplicate action belongs to `Step2Ambientes` (D-06 — "o botão mora em Step2Ambientes, pai que possui o array de ambientes").

**Implementation:** `Step2Ambientes` passes `onDuplicate={() => duplicarAmbiente(i)}` to `AmbienteCard`. `AmbienteCard` renders the button in its header bar, next to the remove button.

**Composition in AmbienteCard header** (lines ~337-340 area):

```tsx
{/* existing remove button */}
<Button size="icon" variant="ghost" className="h-8 w-8 text-destructive hover:text-destructive" onClick={...}>
  <Trash2 className="h-4 w-4" />
</Button>
{/* NEW: Duplicate ambiente button — only rendered when onDuplicate is provided */}
{onDuplicate && (
  <Button
    size="icon"
    variant="ghost"
    className="h-8 w-8 text-muted-foreground hover:text-foreground"
    title="Duplicar ambiente"
    onClick={(e) => { e.stopPropagation(); onDuplicate(); }}
  >
    <Copy className="h-4 w-4" />
  </Button>
)}
```

**Prop added to AmbienteCard:** `onDuplicate?: () => void`

**Clone behavior (D-06, D-08):**
- Deep-clone entire `Ambiente` tree: new UUID for `Ambiente.id`, new UUIDs for every `SistemaIluminacao.id`, every `ItemFitaLED.id`, every `ItemDriver.id`, every `ItemPerfil.id`, every luminária item `id`.
- `nome`: append `" (cópia)"` — e.g. `"Cozinha"` → `"Cozinha (cópia)"`.
- Cloned ambiente inserted at `i + 1` (immediately after original) in the `ambientes` array.
- No dialog, no confirmation (D-07).

**States:**

| State | Appearance |
|-------|-----------|
| Default | `text-muted-foreground` ghost icon button, h-8 w-8 |
| Hover | `text-foreground` |
| Disabled | Not disabled |

### UX-05 — Checklist pré-PDF (Step3Revisao)

**Location:** `Step3Revisao.tsx` — inserted as the **first element** inside the top-level `<div className="space-y-6">` container, before the existing content. Always visible while Step 3 is active.

**Composition:**

```
Card className="border"            ← shadcn Card — no shadow variant needed
  CardHeader className="pb-2"
    CardTitle className="text-sm font-semibold flex items-center gap-2"
      CheckCircle2 | AlertTriangle (lucide, h-4 w-4)  ← icon changes per state
      "Verificação pré-PDF"
  CardContent className="pt-0 space-y-1"
    {/* SUCCESS STATE — no issues */}
    p.text-sm.text-muted-foreground  "Tudo certo! Nenhum item suspeito encontrado."

    {/* OR: list of issues */}
    {issues.map(issue => (
      div.flex.items-center.justify-between.gap-3.py-1  key={issue.id}
        div.flex.items-center.gap-2
          span  {issue.level === 'error' ? '🔴' : '🟡'}
          span.text-sm.text-foreground  {issue.mensagem}
        Button variant="link" size="sm" className="h-auto p-0 text-xs text-primary"
          "corrigir"   ← navigates back to Step 2 (calls onPrev()) — see navigation note
    ))}
```

**Issue levels and messages (D-10, D-11):**

| Level | Trigger condition | Mensagem |
|-------|-------------------|----------|
| 🔴 erro | `sis.fita.codigo && (!sis.metragemManual \|\| sis.metragemManual <= 0)` | `"[AmbienteNome] — Fita sem metragem (0m): o orçamento ficará R$ 0,00"` |
| 🟡 aviso | `sis.fita.codigo && !sis.driver.codigo` | `"[AmbienteNome] — Sistema sem driver"` |
| 🟡 aviso | `sis.driver.codigo && !sis.fita.codigo` | `"[AmbienteNome] — Driver sem fita LED"` |
| 🟡 aviso | `sis.perfil && !sis.fita.codigo` | `"[AmbienteNome] — Perfil sem fita LED"` |
| 🟡 aviso | fita voltagem ≠ driver voltagem (both defined) | `"[AmbienteNome] — Voltagem divergente: fita {X}V × driver {Y}V"` |
| 🟡 aviso | `luminariaPrecisaLampada(lum.descricao) && !ambienteTemLampada(amb)` | `"[AmbienteNome] — Peça sem lâmpada: {lum.descricao}"` |

**Issue ordering:** all 🔴 errors first, then 🟡 warnings, grouped by ambiente.

**Success state (no issues):**

```tsx
<div className="flex items-center gap-2">
  <CheckCircle2 className="h-4 w-4 text-emerald-500" />
  <p className="text-sm text-muted-foreground">Tudo certo! Nenhum item suspeito encontrado.</p>
</div>
```

CardTitle icon when success: `CheckCircle2` with `text-emerald-500`.
CardTitle icon when any issue: `AlertTriangle` with `text-yellow-600` (warnings only) or `text-destructive` (any error present).

**"Gerar PDF" gate (D-12):**

The existing "Gerar PDF" / `FileDown` button in `Step3Revisao` is wrapped with the blocking gate:

```tsx
const temErroBlockante = checklistIssues.some(i => i.level === 'error');

<Button
  onClick={handleGerarPdf}
  disabled={temErroBlockante || savingOrcamento}
  className="gap-2"
>
  <FileDown className="h-4 w-4" />
  Gerar PDF
</Button>
```

When `temErroBlockante` is true, the button is visually `disabled` (shadcn default disabled styles — opacity-50, pointer-events-none). No additional tooltip needed; the 🔴 checklist row already explains the block.

**"corrigir" navigation:** calls `onPrev()` (navigates back to Step 2). No deep-link to specific ambiente is specified — step navigation alone satisfies D-11. If `onPrev` is not in scope from Step3Props, it is already present (line 28 of Step3Revisao.tsx).

**Checklist panel states:**

| State | Card appearance |
|-------|----------------|
| No issues | Green `CheckCircle2` icon, muted success message, no rows |
| Warnings only (🟡) | Yellow `AlertTriangle` icon, list of warning rows with "corrigir" links; PDF button enabled |
| Has blocking error (🔴) | Red `AlertTriangle` (`text-destructive`) icon, error rows first then warning rows; PDF button `disabled` |
| Mixed (errors + warnings) | Red icon, errors first, warnings after; PDF blocked |
