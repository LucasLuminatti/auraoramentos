    Classes:
    - **OK natural** — Tela usada por colab (não-admin) onde RLS filtrar = comportamento desejado. Ex: ArquitetoAutocomplete dentro de cliente → colab vê só os arquitetos dele.
    - **OK admin-only** — Tela só é renderizada para admin (rota /admin protegida ou subseção dela). RLS deixa admin ver tudo via `has_role(admin)`, então comportamento idêntico ao atual.
    - **Risk** — Tela é renderizada por colab E precisa ver dados de outros users. Hoje não deve existir esse caso (arquiteto/cliente sempre filtrado por dono), mas se aparecer, registrar e abrir TODO.
